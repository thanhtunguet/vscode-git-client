# IntelliJ Git Client: Features That Are Worth It

## Daily Workflow Speedups
- **Commit Tool Window**: stage/unstage, partial commits, amend last commit, and commit message templates in one place.
- **Line-level and hunk-level staging**: commit only exact changes you want.
- **Smart changelists**: group unrelated work into separate logical buckets before committing.
- **Shelf/Unshelf changes**: park local work quickly without creating noisy WIP commits.
- **Quick Git actions (`Ctrl/Cmd+Shift+A`)**: run almost any Git operation from one command search.

## Branching and History
- **Visual Git Log & Graph**: inspect branches, merge points, and commit ancestry clearly.
- **Branch popup UX**: create/switch/checkout/track remote branches quickly.
- **Interactive Rebase UI**: reorder/squash/drop/edit commits without memorizing CLI flags.
- **Cherry-pick and revert from log**: apply or undo commits with minimal friction.
- **Reset options with guardrails**: soft/mixed/hard reset from UI with clear context.

## Diff, Review, and Understanding Changes
- **Advanced diff viewer**: side-by-side unified views, syntax-aware highlighting, whitespace options.
- **Inline gutter change markers**: see local edits vs. HEAD instantly while coding.
- **Before/after file history**: track how a file evolved over time.
- **Annotate/Blame view**: find who changed each line and jump to commit details.

## Merge and Conflict Resolution
- **3-way merge tool**: structured conflict resolution with left/right/base context.
- **Conflict navigation shortcuts**: move through conflicts quickly in big merges.
- **Automatic conflict detection in workflow**: surfaced directly in IDE, not hidden in terminal output.

## Remote and Collaboration Features
- **Push/Pull with preview**: inspect outgoing/incoming commits before syncing.
- **Fetch and prune support**: keep local refs clean and current.
- **Integrated Pull Request workflows** (GitHub/GitLab plugins): review comments, diffs, and discussions in IDE.
- **Issue/task integration**: tie commits/branches to issue IDs and task context.

## Quality and Safety Nets
- **Pre-commit checks**: run inspections/tests/formatters before commit.
- **Commit checks per file/changelist**: reduce accidental broken commits.
- **Local History (IDE feature)**: extra safety net even outside Git commits.
- **Undo-friendly UI flows**: lower risk for occasional Git users.

## Productivity Extras (IntelliJ-specific)
- **Code-aware merge and diff context**: better readability for large refactors.
- **Unified experience with refactoring tools**: Git + refactor + inspections in one loop.
- **Keyboard-first + UI-first parity**: good for both power users and teams with mixed Git skill levels.

## Why Teams Usually Find It “Worth It”
- Fewer context switches between editor and terminal.
- Lower Git mistake rate for non-expert contributors.
- Faster review and conflict resolution cycles.
- Better visibility into history and branch state for everyone.
